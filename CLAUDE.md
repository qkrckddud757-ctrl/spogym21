# FitConsult — 헬스장 PT 1:1 컨설팅 시스템

## 한 줄 요약
PT 트레이너 한 명 한 명에게 AI가 "1:1 컨설팅 느낌"의 데일리 할 일·회원별 재등록 멘트·주간 전략을 짜주는 내부용 웹 시스템. 본인 지점(15명 규모)부터 시작해 추후 타 지점 확장.

## 기술 스택 (고정)
- **프론트·서버**: Next.js (App Router)
- **코드**: GitHub
- **배포**: Vercel
- **DB·Auth·권한**: Supabase (Postgres + Auth + Row Level Security)
- **로그인**: 구글 OAuth (Supabase Auth 경유)
- **AI**: **Google Gemini** (`@google-cloud/vertexai` — Vertex AI 경유). 기본 모델 `gemini-2.5-flash` (저비용·빠름)
- 이 스택 밖은 꺼내지 않음. 필요하면 라이브러리·API 수준에서만 추가.

## 핵심 원칙
1. **1단계는 "돌아가는 최소 버전"** — 배포 URL에서 화면이 실제로 떠야 2단계 시작. 기능 전에 인프라.
2. **욕심 금지** — 회원권 부서·SNS 마케팅 자동생성·멀티 지점은 MVP 범위 밖. 꺼내지 말 것.
3. **플랜 먼저, 실행은 허락 받고** — 사용자가 명시적으로 지시하기 전엔 파일 수정·실행 금지.
4. **권한은 DB 레벨에서 막는다** — 가시성 규칙은 Supabase RLS로 강제. 앱 코드 버그 있어도 못 뚫게.
5. **MVP의 "심장"은 4단계** — PT 회원 1명에 대해 Gemini Flash가 재등록 멘트 초안을 생성하는 순간. 그 전까지는 모두 토대 작업.
6. **AI 출력 톤 고정** — 심리학 기반 세일즈, 거부감 없이 스스로 설득되는 구조, 즉시 실행 가능한 현실적인 멘트.

## 개발 순서 개요 (6단계 + 선택 1)
1. Next.js 돌아가는 최소 버전 (Vercel 배포까지)
2. Supabase 연결 + 구글 OAuth 로그인 + `profiles` 테이블
3. PT 회원 CRUD + 직급 기반 RLS 정책
4. **Gemini API** 붙여서 AI 재등록 멘트 생성 (핵심)
5. 엑셀 업로드 + OT·체험 명단 + 개인별 지표 계산
6. 주간 전략 리포트 + 팀장·매니저 집계 뷰
7. (선택) 구글 드라이브 "제로코치 자료" RAG 연동

## 오늘 안 할 것 (욕심 제거)
- 회원권 부서 기능 (장기미방문·상담 미등록 관리 등)
- SNS(당근마켓·네이버체험) 마케팅 자료 자동생성
- 멀티 지점·본사 계층 구조
- RAG / 구글 드라이브 연동 (앞 6단계 끝낸 뒤)
- 복잡한 UI 꾸미기 (tailwind 기본 스타일만, 컴포넌트 라이브러리 추가 X)
- 알림·이메일·푸시 발송
- 결제·정산 기능
- 한 번에 두 단계 이상 건너뛰기
